Приоритет 4. Metadata cleanup

Цель
- закрыть оставшиеся проблемы с пустыми и слишком длинными metadata
- привести важные URL к предсказуемым сниппетам
- не дать theme fallback снова раздувать title и description

Почему это уже не первый приоритет
- проблема живая, но уже не массовая как в старом аудите
- сейчас сначала важнее главная, ключевые collections и multiple H1

Исходные проблемы
- пустых meta description: 6 URL
- длинных title больше 65 символов: 12 URL
- длинных meta description больше 160 символов: 21 URL

URL без meta description
- https://mamapack.pl/blogs/news
- https://mamapack.pl/blogs/przydatne-artykuly
- https://mamapack.pl/collections/formam
- https://mamapack.pl/collections/forbaby
- https://mamapack.pl/pages/contact
- https://mamapack.pl/collections/zabawa-i-edukacja

С кого начинать
1. Главная.
2. gotowe-torby, formam, forbaby.
3. Остальные коммерческие collections.
4. Contact page и blog index pages.
5. Статьи блога.

Блок 1. Пустые meta description

Что делать
1. Для каждой страницы вручную заполнить Search engine listing preview в Shopify admin.
2. Не оставлять пустое поле в расчете на theme fallback.
3. Если тема все равно строит description из контента, ограничить fallback.
4. Если description приходит автоматически, проверить, не перетирает ли он ручной вариант.

Правило написания description
1. Один главный запрос.
2. Одна выгода.
3. Одно уточнение по странице.
4. 140-170 символов.
5. Никаких обрывков текста из description.

Блок 2. Длинные title

Что проверить
1. Title собирается вручную или шаблоном.
2. Где именно раздувается длина:
- слишком длинное название страницы
- автоматическое добавление бренда
- повтор характеристик
- дублирование типа товара и атрибутов

Как сокращать title
1. Оставить главную ключевую фразу.
2. Убрать второстепенные атрибуты.
3. Бренд сайта оставлять только если длина позволяет.
4. Не пытаться втиснуть все характеристики товара в title.

URL, на которые обратить внимание в первую очередь
- https://mamapack.pl/
- https://mamapack.pl/collections/oslonki-laktacyjne
- https://mamapack.pl/collections/jednorazowe-majtki
- https://mamapack.pl/collections/higieniczne-wkladki
- длинные статьи из blogs/przydatne-artykuly

Блок 3. Длинные meta description

Что обычно вызывает проблему
1. Theme fallback берет длинный кусок контента.
2. Description импортирован как длинный лид.
3. Description дублирует вступление статьи почти целиком.
4. В description пытаются перечислить слишком много выгод.

Как сокращать description
1. Убрать перечисление всех характеристик.
2. Убрать повтор H1.
3. Убрать воду и длинные вводные конструкции.
4. Оставить коммерческий смысл в одном предложении.
5. Для статьи оставить тему и обещание полезности, но без полного лида.

Правила по типам страниц

Главная
- общий коммерческий запрос
- выгода
- формат предложения

Collections
- запрос категории
- что внутри категории
- чем удобен выбор

Products
- не перечислять все характеристики
- оставлять тип товара, главное преимущество и CTA-смысл

Blog articles
- тема статьи
- кому полезна
- какой вопрос закрывает

Contact page
- короткое описание того, как связаться с магазином

Blog index pages
- короткое описание тематики блога и пользы статей

Техническая часть
1. Проверить, выводит ли тема page_description как есть.
2. Если fallback нужен, ограничить его длину.
3. Не тянуть в meta description raw HTML или длинный description body.
4. Проверить, нет ли конфликтов между ручными SEO-полями и шаблонным выводом.

Шаблон рабочей очереди
1. URL.
2. Тип страницы.
3. Текущий title.
4. Новый title.
5. Текущий description.
6. Новый description.
7. Источник изменения: админка или тема.
8. Статус проверки после публикации.

Проверка после изменений
1. В HTML правильный title.
2. В HTML правильный description.
3. Description не пустой.
4. Description не обрезается на середине слова.
5. Длина title и description в пределах цели.

Критерий готовности
- у важных URL нет пустых description
- длинные metadata сокращены в первую очередь на коммерческих страницах
- тема больше не генерирует случайно длинные сниппеты
