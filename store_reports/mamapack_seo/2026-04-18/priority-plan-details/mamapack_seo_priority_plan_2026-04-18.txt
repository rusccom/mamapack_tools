Приоритет 2. Исправить H1-логику на product pages и убрать мусор из описаний

Почему это приоритет номер 2
- проблема живая и массовая: 18 URL с несколькими H1 и 2 URL с пустым H1
- часть дублирующих H1 приезжает из product description, а часть из шаблона
- это системная HTML-проблема, которую лучше убрать до масштабирования контента

Что сделать
1. Пройтись по product template и убедиться, что на странице остается ровно один H1.
2. Проверить, не выводится ли H1 вторично внутри title-блока, карточки, sticky-блока или custom section.
3. Запретить попадание H1 из product description в видимую структуру страницы.
4. Почистить product descriptions от вставленных h1/h2, где они пришли из импортов.
5. Для пустого H1 на главной и товарах найти источник пустого тега и удалить его.
6. Проверить после правки не только 3-5 URL вручную, а всю sitemap-выборку.

Приоритет 3. Создать карту SEO-посадок без каннибализации и только потом запускать новые URL

Почему это приоритет номер 3
- новые SEO-страницы действительно нужны
- но запуск без карты интентов легко создаст дублирование между gotowe-torby, wyprawka-do-szpitala, torba-do-porodu, formam и forbaby

Что сделать сначала до создания новых collections
1. Собрать список ключевых коммерческих запросов.
2. Для каждого запроса определить:
- главную целевую страницу
- вторичную страницу поддержки
- чем страница отличается от соседних
- какие товары на ней должны быть
- какие внутренние ссылки она должна получать
3. Отдельно решить судьбу текущих страниц:
- gotowe-torby
- formam
- forbaby
4. Для каждой текущей страницы выбрать только один сценарий:
- оставляем и ретаргетим
- объединяем с новой страницей и ставим 301
- оставляем как вспомогательную страницу с другим интентом
5. Только после этой карты создавать новые collections.

Рекомендуемый стартовый набор посадок
- /collections/wyprawka-do-szpitala
- /collections/torba-do-porodu
- /collections/gotowa-torba-do-szpitala

Что пока не делать без карты интентов
- не создавать сразу 5 почти одинаковых категорий
- не плодить страницы с одинаковым набором товаров и похожими title/H1
- не ставить массовые 301 без решения, какая страница станет основной

Минимум, который должен быть у каждой новой landing page
- 1 запрос = 1 главный intent
- title 50-65 символов
- meta description 140-170 символов
- 1 H1
- лид 60-100 слов
- 250-500 слов полезного текста
- FAQ на 4-6 вопросов
- внутренняя перелинковка на соседние категории и блог

Кто делает
- SEO
- контент
- админка Shopify
- при необходимости тема Shopify

Критерий готовности приоритета 3
- есть таблица интентов по коммерческим запросам
- для каждой важной collection уже определена роль
- новые страницы создаются без пересечения по смыслу

Приоритет 5. Усилить внутреннюю перелинковку и навигацию

Почему это приоритет номер 5
- после уточнения семантики collections надо направить вес сайта в нужные страницы
- сейчас без нормальной перелинковки даже хорошие новые посадки будут расти медленнее

Что сделать
1. С главной дать текстовые ссылки на 4-6 приоритетных коммерческих categories.
2. Из каждой важной category ссылаться на 2-4 соседние categories.
3. На collections добавить блок Powiązane kategorie.
4. На product pages добавить ссылку на основную SEO-category.
5. Из статей блога вести ссылки на коммерческие categories.
6. Добавить хлебные крошки на category и product pages, если их нет в удобном виде для пользователя.
7. Проверить BreadcrumbList только после того, как сами хлебные крошки реально видимы на странице.

Кто делает
- тема Shopify
- контент

Критерий готовности приоритета 5
- у главной есть текстовые ссылки на ключевые collections
- у collections есть блоки связанной навигации
- у products есть ссылка на главную категорию
- у блога есть ссылки на коммерческие страницы

Приоритет 6. Усилить контентный кластер под коммерческие категории

Почему это приоритет номер 6
- у сайта мало контентных URL для устойчивой поддержки коммерческих страниц

2. Каждую статью писать под отдельный информационный запрос, но с выходом в коммерческий intent.
3. В каждой статье ставить 3-5 ссылок на приоритетные collections.
4. На collections ссылаться обратно на релевантные статьи.

Приоритет 7. Ввести режим повторной SEO-валидации после каждого спринта

Почему это важно
- старый план устарел за несколько дней
- без повторного mini-crawl приоритеты быстро перестают соответствовать реальности

Что делать после каждого блока работ
1. Повторно прогонять sitemap.xml.
2. Снимать:
- пустые meta description
- длинные title
- длинные meta description
- пустые H1
- multiple H1
- наличие Product schema на product pages
3. Обновлять backlog только по живым проблемам.
4. Не тратить новый спринт на уже закрытые задачи.

Рекомендуемый порядок работ на ближайшие 14 дней
1. Починить главную: непустой H1, title, description, intro, ссылки на collections.
2. Переписать gotowe-torby, formam и forbaby под спрос.
3. Исправить multiple H1 и пустые H1 на product pages.
4. Заполнить пустые meta description и сократить оставшиеся длинные metadata на приоритетных URL.
5. Собрать карту интентов для новых SEO-landing pages.
6. Только после карты интентов создать 2-3 новые посадки, а не 5 сразу.
7. Добавить перелинковку между главной, collections, products и блогом.

Что я бы делал первым, если цель - рост органики быстрее всего
1. Исправил бы главную, потому что она уже ранжируется и может быстрее всего передавать вес.
2. Переделал бы gotowe-torby под запрос gotowe torby do szpitala.
3. Сделал бы formam и forbaby полноценными коммерческими collections.
4. Убрал бы multiple H1 на товарах.
5. Потом только запускал бы новые SEO-посадки через карту интентов.
