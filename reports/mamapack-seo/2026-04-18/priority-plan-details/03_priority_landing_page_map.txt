Приоритет 3. Карта SEO-посадок и защита от каннибализации

Цель
- спланировать новые collection pages так, чтобы каждая закрывала отдельный коммерческий intent
- не плодить похожие страницы с одинаковым набором товаров
- заранее решить, какие существующие URL сохраняются, а какие объединяются

Почему это нельзя делать без подготовки
- у сайта уже есть gotowe-torby, formam и forbaby
- если сразу создать wyprawka-do-szpitala, torba-do-porodu, wyprawka-dla-mamy, wyprawka-dla-noworodka и gotowa-torba-do-szpitala, часть страниц неизбежно начнет конкурировать между собой
- новые collections без карты интентов создадут внутреннее размывание сигналов

Главное правило
- один главный запрос = одна главная SEO-страница

Что собрать до запуска новых URL
1. Список ключевых коммерческих запросов.
2. Частотность или хотя бы приоритетность запросов.
3. Поисковый intent по каждому запросу.
4. Пересечения между запросами.
5. Текущие URL сайта, которые уже частично отвечают на этот спрос.

Шаблон карты интентов
Для каждого запроса зафиксировать:
1. Запрос.
2. Тип intent.
3. Главная целевая страница.
4. Вспомогательная страница.
5. Почему страница отличается от соседних.
6. Какие товары должны быть на странице.
7. Какие товары нельзя выводить на странице.
8. Какие внутренние ссылки нужны.
9. Нужен ли 301 со старого URL.
10. Нужен ли noindex для вспомогательного фильтра или дубля.

Стартовый список коммерческих запросов
- wyprawka do szpitala
- torba do porodu
- gotowa torba do szpitala
- wyprawka dla mamy do szpitala
- wyprawka dla noworodka do szpitala

Как разделять intent между страницами

Страница wyprawka-do-szpitala
- это самая широкая коммерческая категория
- она должна закрывать общий спрос
- сюда можно вести пользователя, который ищет полный комплект или категорию в целом

Страница torba-do-porodu
- это страница про сам формат torba и момент wyjazdu do szpitala
- здесь важны готовые комплекты, удобство упаковки, список вещей на poród

Страница gotowa-torba-do-szpitala
- это еще более прямой коммерческий запрос про готовое решение
- здесь акцент на gotowe zestawy, oszczędność czasu, gotowe warianty

Страница formam
- это не общая выправка
- это сегмент только для вещей мамы
- сюда не должны бездумно попадать все детские товары

Страница forbaby
- это не общая выправка
- это сегмент только для rzeczy dla noworodka
- сюда не должны бездумно попадать товары чисто для mamy

Решение по существующим URL нужно принять до запуска новых

Вариант A. Оставить gotowe-torby как главную страницу под gotowe torby do szpitala
- тогда не создавать вторую почти идентичную страницу
- можно только переименовать handle, если это оправдано и если есть готовность ставить 301

Вариант B. Создать новый URL gotowa-torba-do-szpitala
- тогда старую gotowe-torby нужно либо 301 редиректить, либо оставлять под более широкую категорию
- нельзя оставлять две страницы с одинаковым набором товаров и одинаковым текстом

Вариант C. Оставить gotowe-torby как ассортиментную категорию
- а wyprawka-do-szpitala сделать широкой SEO-страницей с подборкой
- тогда должны различаться title, H1, текст, товары и внутренняя перелинковка

Что рекомендую как безопасный старт
1. Сначала ретаргетить gotowe-torby, formam и forbaby.
2. Потом создать только 2-3 новые страницы:
- wyprawka-do-szpitala
- torba-do-porodu
- при необходимости gotowa-torba-do-szpitala
3. Остальные страницы запускать только после оценки, хватает ли различий по intent.

Как понять, что страницы пересекаются слишком сильно
1. У них почти одинаковые H1.
2. У них почти одинаковые title.
3. У них одинаковый набор товаров.
4. У них одинаковые FAQ.
5. У них одинаковые internal links.
6. Они отвечают на один и тот же пользовательский вопрос.

Как избежать каннибализации
1. Для каждой collection задать один главный запрос и 2-3 вторичных.
2. Не использовать одну и ту же ключевую фразу как главный H1 на двух страницах.
3. Разводить страницы по набору товаров и по текстовому углу.
4. Делать разные хлебные крошки и разную внутреннюю перелинковку.
5. Если новая страница полностью сильнее старой, ставить 301.

Как принимать решение про 301 redirect
1. Если старая страница и новая страница закрывают один и тот же intent, нужна одна основная страница и 301 со второй.
2. Если страницы отличаются по intent, 301 не нужен.
3. Нельзя делать 301 просто потому, что новый handle красивее.
4. До редиректа нужно обновить внутренние ссылки.

Что должно быть у каждой новой collection page
1. Уникальный handle.
2. Один H1.
3. Один главный query target.
4. Title 50-65 символов.
5. Description 140-170 символов.
6. Лид 60-100 слов.
7. Основной текст 250-500 слов.
8. FAQ 4-6 вопросов.
9. Явные ссылки на соседние коллекции и статьи.

Практический рабочий шаблон файла планирования
1. URL.
2. Query target.
3. H1.
4. Title.
5. Description.
6. Список товаров.
7. Соседние страницы.
8. Нужен ли redirect.
9. Нужен ли новый контент.
10. Нужна ли новая тема или хватает Shopify admin.

Кто делает
- SEO strategist
- Shopify admin manager
- content manager
- theme developer при необходимости новых шаблонных блоков

Порядок выполнения
1. Собрать карту интентов.
2. Утвердить роли существующих страниц.
3. Утвердить, какие новые URL реально нужны.
4. Подготовить тексты и metadata для каждой новой страницы.
5. Обновить внутренние ссылки.
6. Только потом публиковать новые collections.
7. После публикации смотреть индексацию и поведение в Search Console.

Критерий готовности
- по каждому коммерческому запросу есть одна главная SEO-страница
- нет двух collections с одинаковым intent
- все новые URL создаются осознанно, а не по принципу чем больше, тем лучше
