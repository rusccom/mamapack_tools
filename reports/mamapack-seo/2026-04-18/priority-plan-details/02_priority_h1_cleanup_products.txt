Приоритет 2. H1-логика на product pages и очистка описаний

Цель
- убрать множественные H1 и пустые H1
- сделать HTML-структуру product pages стабильной
- отделить контентные заголовки от мусорных заголовков, приехавших из шаблона или description

Почему это важно
- multiple H1 подтверждены на живом сайте
- часть H1 дублируется дословно
- часть H1 приезжает из импортированного HTML в description
- пока это не исправлено, product pages выглядят как технически неаккуратные документы

Что считается проблемой
1. На странице больше одного H1.
2. На странице есть пустой H1.
3. На странице H1 появляется не из главного product title, а из description.
4. Внутри description есть h1 или h2, которые ломают структуру документа.

Примеры URL для первичного разбора
- https://mamapack.pl/products/me-figi-l-bialy
- https://mamapack.pl/products/akuku-a0307-szczotka-i-grzebien-z-naturalnym-wlosiem-biala
- https://mamapack.pl/products/szlafrok-damski-z-delikatnego-muslinu-l-wiazany-doctor-nap
- https://mamapack.pl/products/akuku-a0458-zestaw-manicure-rozowy
- https://mamapack.pl/products/akuku-a0306-laktator-reczny

Типовые причины multiple H1
1. Логотип или header section размечены как H1.
2. Product title рендерится дважды в основном блоке и в sticky summary.
3. Product description вставлена из внешнего источника с собственными h1.
4. Rich text section на product template рендерит heading как H1.
5. Custom liquid block отдает второй заголовок товара.

Фаза 1. Диагностика в теме

Что проверить
1. Какая секция рендерит основной product title.
2. Есть ли sticky product summary или mobile block с тем же заголовком.
3. Есть ли rich text block, который допускает H1.
4. Есть ли header logic, где логотип становится H1 на всех страницах.
5. Нет ли condition, где product vendor или special offer block рендерятся как H1.

Что зафиксировать
1. Один файл или section, который должен быть единственным источником H1.
2. Все остальные места, где H1 надо запретить.
3. Какие страницы ломаются шаблоном, а какие ломаются контентом.

Фаза 2. Исправление темы

Базовые правила
1. На каждой product page должен быть ровно один H1.
2. Единственный H1 должен быть главным названием товара.
3. Все вторичные заголовки должны быть H2 или H3.
4. Логотип не должен быть H1 на product pages.
5. Дополнительные секции не должны иметь режим H1.

Что сделать в теме
1. Найти главный product heading и оставить только его как H1.
2. Во всех дублирующих блоках сменить H1 на div, p, h2 или h3 в зависимости от смысла.
3. Если heading level задается настройкой section, исключить H1 для product template.
4. Если sticky block нужен визуально, оставить тот же текст без H1.
5. Если есть условная логика для home page H1, убедиться, что она не протекает на products.

Фаза 3. Очистка контента в админке

Что искать в product descriptions
1. Теги h1.
2. Теги h2, которые вставлены как рыночный мусор и не несут структуры.
3. Внешний HTML из маркетплейсов.
4. Избыточные caps-lock заголовки.
5. Повтор названия товара в самом начале description.

Что делать
1. Удалять h1 из description полностью.
2. H2 оставлять только там, где они реально структурируют полезный текст.
3. Если description короткий и заголовки там не нужны, переводить все в абзацы и списки.
4. Очищать мусорный HTML при импортах.
5. Если у товара очень короткое описание, не пытаться делать искусственную иерархию заголовков.

Очередность ручной чистки
1. Товары с 3 H1.
2. Товары с 2 H1.
3. Товары с пустым H1.
4. Остальные товары, которые импортируются из того же источника.

Как понять, где проблема - в шаблоне или в описании
1. Если второй H1 повторяет product title, это почти всегда тема.
2. Если второй H1 выглядит как рекламный текст или название блока, это либо rich section, либо HTML из description.
3. Если второй H1 относится к характеристикам товара, это почти всегда вставка из описания.
4. Если H1 пустой, чаще всего это пустое theme field или heading block без текста.

Шаблон проверки каждого проблемного URL
1. Открыть HTML страницы.
2. Посчитать H1.
3. Зафиксировать тексты всех H1.
4. Найти их источник в шаблоне или description.
5. Исправить причину.
6. Перепроверить страницу после публикации.

Что не делать
- не лечить проблему CSS-скрытием H1
- не оставлять второй H1 только потому, что он визуально красиво выглядит
- не хранить H1 в product description
- не ставить H1 в FAQ-блоках и feature-блоках

Кто делает
- theme developer
- Shopify content/admin manager

Зависимости
- желательно не менять одновременно смысл product title и H1-логику
- сначала убрать HTML-проблему, потом уже улучшать коммерческие тексты

Порядок выполнения
1. Найти единый источник H1 в шаблоне.
2. Убрать вторичные H1 в theme sections.
3. Пройтись по проблемным товарам в админке и очистить description.
4. Перепроверить список проблемных URL.
5. После этого прогнать всю sitemap product pages повторно.

Проверка качества
1. На любой product page только один H1.
2. Этот H1 совпадает с основным названием товара.
3. Внутри product description нет H1.
4. Внутри product description нет мусорных заголовков, которые ломают структуру.
5. Sticky blocks и promo blocks не создают новых H1.

Критерий готовности
- 0 product URL с несколькими H1
- 0 product URL с пустым H1
- описания очищены от шаблонного и импортного HTML, который ломает заголовки
